# Accounting Service API 規格書 (v2.2)

本專案為 Home Service Hub 的會計微服務，負責處理交易紀錄、信用卡管理、固定支出 (訂閱/分期) 以及財務報表分析。

## 核心技術棧
- **Framework**: FastAPI (Python 3.13)
- **ORM**: SQLAlchemy
- **Database**: PostgreSQL
- **Audit**: 所有主要模型皆繼承 `TimestampMixin`，包含 `created_at` 與 `updated_at` 欄位。

---

## 1. 資料庫模型與欄位詳細說明

### 1.1 交易紀錄 (transactions)
記錄所有收入與支出。支援與訂閱、分期、卡片及分類的關聯。

| 欄位名 | 類型 | 說明 |
| :--- | :--- | :--- |
| `id` | Integer | PK, 自動遞增 |
| `date` | Date | 交易日期 |
| `category` | String | 分類名稱 (冗餘存儲，與 `category_id` 同步) |

| `item` | String | 品項名稱 |
| `personal_amount` | Integer | 個人負擔金額 (TWD) |
| `actual_swipe` | Integer | 實際刷卡金額 (TWD) |

| `card_id` | Integer | FK: `credit_cards.id`, 資金來源信用卡 ID |
| `transaction_type` | String | `EXPENSE` (支出), `INCOME` (收入) |
| `note` | String | 備註 (可選) |
| `tags` | JSON | 標籤陣列 (可選) |
| `status` | String | `COMPLETED` (已完成), `PENDING_SUB` (訂閱生成), `PENDING_INSTALLMENT` (分期生成) |
| `related_transaction_id`| Integer | FK: `transactions.id`, 原始交易 ID (退款/沖銷連結) |
| `subscription_id` | Integer | FK: `subscriptions.id`, 來源定期項目 ID |
| `installment_id` | Integer | FK: `installments.id`, 來源分期計畫 ID |

## 2. 核心邏輯 (核心非 CRUD 處理)

### 2.1 智慧資料校驗與自動路由 (Transaction Service)
當執行 `create` 或 `update` 交易時：
- **Category Sync**: 若提供 `category_id`，系統強制查詢 `categories` 表並覆寫 `category` 文字欄位。
- **Auto Routing**: 若 `card_id` 為空且 `payment_method` (如 "Line Pay") 存在於 `payment_routes` 中，系統會自動填寫對應的 `card_id`。
- **Card Sync**: 若傳入 `card_id` 但支付工具為空或預設為 "信用卡"，系統會自動將 `payment_method` 設為該卡片名稱 (向後相容)。
- **ID Validation**: 非法 `category_id` 或 `card_id` 會觸發 `400 Bad Request`。

### 2.2 信用卡回饋週期計算 (Billing Service)
- **日曆月模式**: 統計範圍 = 當月 1 號 ~ 當月最後一天。
- **帳單週期模式**: 根據 `billing_day` 判斷。範圍為結帳日隔天至下次結帳日。

### 2.3 定期帳生成機制 (Recurring Service)
- 系統會比對 `subscriptions` 與 `installments` 是否在「當月」已有對應的交易紀錄。
- **類型區分**: 分為固定支出 (`FIXED_EXPENSE`) 與數位訂閱 (`SUBSCRIPTION`)，方便前端分開管理與後續報表拆分。

---

## 3. 審計與元數據
- **Audit Columns**: 所有表（除路由表）均包含 `created_at` 與 `updated_at`。
- **Integer Amounts**: 系統全面改用整數處理金額，不支援小數點。
