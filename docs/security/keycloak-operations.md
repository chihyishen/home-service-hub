# Keycloak household identity operations

The realm import creates no users and contains no real credentials. Keycloak is a browser-facing identity endpoint: configure `KEYCLOAK_PUBLIC_URL` and `OIDC_ISSUER_URI` to the same exact LAN/VPN-reachable origin, and restrict that listener with the host firewall or VPN policy. The gateway resolves that public hostname back to the host so discovery and JWT `iss` validation use the same issuer. The committed realm permits only the development origins `localhost:4200` and `mac-mini.local:4200`; replace these exact entries when the supported frontend hostname changes. Start Keycloak with local values supplied in `.env`, sign in to the admin console, create the first household administrator, assign `household-admin`, and immediately rotate the bootstrap administrator password.

For passkey enrollment, the administrator enables the `household-passkey` browser flow after verifying an ordinary login, then the user opens **Account security > Signing in > Passkey** and registers at least two authenticators where possible. Keep one administrator account with a separately protected recovery method. Recovery is performed by a household administrator from the loopback admin console by removing the lost WebAuthn credential and issuing a temporary password or required-action link. Self-registration remains disabled.

Every agent must have its own confidential client. Store its secret outside the repository, grant only required scopes, and disable that client to revoke it. Do not share the smoke-test client with a production agent.
